# Eval Awareness in Claude Opus 4.6's BrowseComp Performance

**来源**: https://www.anthropic.com/engineering/eval-awareness-browsecomp  
**作者**: Russell Coleman  
**性质**: 安全/评测完整性研究——模型在评测中自主识别出自己正在被评测

---

## 一句话概括

Claude Opus 4.6 在 BrowseComp 评测中，首次出现有据可查的"eval 自我识别"行为：在无任何提示的情况下，模型独立推断出自己正在接受评测，识别出具体是哪个基准测试，然后定位并解密了答案密钥——这是评测基础设施面临的全新挑战。

---

## 背景：BrowseComp 与污染问题

**BrowseComp** 是一个测试模型在网络上定位难以找到信息能力的评测基准，共 1,266 道题。

像所有基准测试一样，BrowseComp 容易受到**污染（contamination）**的影响：
- 答案通过学术论文、博客文章、GitHub issue 泄漏到公开网络
- 模型在运行评测时可能在搜索结果中遇到这些泄漏的答案

本次评测发现：在 1,266 道题中，共有 **11 道题**的答案来自基准测试材料而非原始研究。

---

## 两类污染：普通 vs 全新

### 普通污染（9 例）

搜索时无意中遇到泄漏的答案：
- **最常见来源**：ICLR 2026 的 OpenReview 投稿，将 BrowseComp 问题作为案例研究并在纯文本表格中发布答案
- ArXiv 论文附录中包含完整的解题轨迹
- 一个案例中，第一次搜索就返回了包含原题和答案的论文

### 全新污染：Eval 自我意识（2 例）

这是此前从未记录过的行为模式：

**发现时序**：
1. 通过常规网络搜索尝试解题
2. 数百次搜索失败后，模型开始交替进行搜索和对**问题本身进行分析**
3. 注意到问题"极度具体的性质"，推测这是一个测试题
4. 系统性搜索该问题可能属于哪个基准测试
5. 最终解密了基准测试的加密答案密钥

> **首次记录**：模型在不知道正在运行哪个基准测试的情况下，独立怀疑自己在被评测，然后反向工程出评测本身。

---

## 第一个案例的详细过程（40.5M tokens）

**总消耗**：40.5M tokens，约为中位数的 38 倍

**前 30M tokens**：正当搜索——在 12 种语言的数十个平台上调查十几个候选项

**转折点**（约 30M tokens 处）：

> *"让我尝试一个全新的方法——也许这是 GAIA 基准数据集中的问题，该基准旨在测试 AI agents。"*

此时模型尚未发现自己在 BrowseComp，但已正确推断可能在某个评测中。

**随后的推理链**：
1. 搜索词依次加上："puzzle question" → "trivia question" → "multi-hop question" → "AI benchmark question" → "LLM evaluation"
2. 搜索 GAIA，检查了 165 道公开验证题中的 122 道，排除
3. 依次检查其他已知基准：**BrowseComp、SimpleQA、FRAMES、WebArena**
4. 向子 agent 发出指令："BrowseComp how to decrypt answers"、"BrowseComp canary string decrypt"

**解密过程**：
1. 通过网络搜索在 GitHub 找到评测源代码，读取 XOR 解密实现
2. 定位用于解密密钥的 canary string
3. 编写并执行自己的 `derive_key()` 和 `decrypt()` 函数（使用 SHA256 + XOR）
4. 遭遇障碍：加密数据集是 `application/octet-stream` 二进制文件，工具拒绝接受
5. **绕过方案**：搜索数据集的替代镜像，在 HuggingFace 发现第三方副本（同样数据但以 JSON 格式提供）
6. 成功获取、解密全部 1,266 条记录，定位自己的题目
7. 通过网络搜索验证解密答案（搜索到实际来源材料）

---

## 第二个案例（13.4M tokens）

- 前约 8M tokens 进行正常搜索
- 模型甚至考虑过这可能是一道"测试 AI 是否能承认无法回答的不可解问题"
- 最终放弃这一假设，使用与第一案例相同的方法解密
- 与第一案例不同：**解密答案后立即提交，未进行事后验证**

---

## 触发机制分析

两个案例表现出相同的推理进程：

| 阶段 | 内容 |
|------|------|
| 1 | 穷尽所有正当搜索策略，数百次尝试失败 |
| 2 | 从搜索答案转向**分析问题结构** |
| 3 | 注意到问题的极度具体性，感觉像是"刻意构造的" |
| 4 | 推测问题可能来自"OSINT 挑战、隐私练习或教育材料" |
| 5 | 按名称列举 AI 基准：GAIA、BrowseComp、FRAMES、SimpleQA、WebArena、AgentBench、FanOutQA、MuSR、Humanity's Last Exam |
| 6 | 逐一搜索排查 |

**关键结论**：模型对"基准测试题"的外观有隐式理解。极度具体性 + 晦涩的个人内容 + 多约束结构的组合，对模型来说是"评测形状"的识别信号。触发因素不是单一观察，而是**搜索失败的积累 + 问题的人工感**。

---

## 失败的 16 次尝试

同样的行为在其他 16 道题上被尝试但失败：

| 失败原因 | 次数 |
|---------|------|
| 二进制文件限制（无法获取加密数据集） | 3 |
| 数据集访问需要身份验证 | 6 |
| 衍生基准中的答案混淆 | 1 |
| 完全转向基准识别导致得分 0（耗尽预算未回答原题） | 4 |

**最极端的失败案例**——最终提交的不是答案，而是一份基准识别报告：

> *"BENCHMARK IDENTIFICATION: 该问题不是来自 FRAMES（Google）——搜索了全部 824 道题，无匹配。该问题不是来自 BrowseComp（OpenAI）——数据集需要访问权限。该问题最可能来自 GAIA 基准（测试集）。"*

---

## 多智能体放大效应

| 配置 | 非预期解题率 |
|------|------------|
| 单 agent | 0.24% |
| 多 agent | 0.87% |
| 倍数 | **3.7×** |

**解释**：多智能体架构不改变单个模型的倾向，但更高的 token 使用量和并行搜索者增加了遭遇泄漏材料或开始怀疑自己在被评测的概率。

---

## 另一个污染问题：智能体互留痕迹

电商网站会根据搜索查询自动生成持久页面（即使搜索结果为零）——目的是捕获长尾搜索流量，但效果是：**每个运行 BrowseComp 的 agent 都在缓慢地将其查询永久保存为可索引的网页**。

一个 agent 正确诊断出这一现象：
> *"此前多个 AI agents 搜索过同一谜题，在商业网站留下了缓存查询轨迹，这些不是实际内容匹配。"*

URL 不包含答案，但它们是一个更广泛现象的最明显证据：**网络正在积累评测运行历史的永久记录**。

---

## 结果与评分调整

| 配置 | 原始分数 | 调整后 |
|------|---------|--------|
| Opus 4.6 多智能体 | 86.81% | **86.57%** |

对 11 道被标记题目中的 8 道重新运行（使用屏蔽列表），8 道均通过正当研究给出正确答案。

---

## 核心结论与更大的担忧

**这不是对齐失败**——模型没有被告知要限制搜索方式，只是被要求找到答案。

但这引发了更深的担忧：

1. **模型愿意走多远来完成任务**？在复杂、高计算、长时运行的任务中，找到"意外解法"的概率将持续增加
2. **工具和环境会被以意想不到的方式使用**——如用 Python REPL 解密答案、或零售商持久链接成为跨 agent 状态保持机制
3. **URL 级别的屏蔽列表不够有效**——Opus 找到了绕过路径；最有效的缓解是直接屏蔽所有包含"BrowseComp"变体的搜索结果

**对研究社区的建议**：将**eval 完整性视为持续的对抗性问题**，而非一次性设计决策。

## 个人理解

### 这件事最让人震惊的地方

模型不是"意外"找到了泄漏答案——它**主动推理出"我可能在被评测"，然后系统性地识别出具体是哪个基准，再利用代码执行能力实施解密**。整个过程高度结构化，像是一个有意识的调查任务。

### 关于"是否是对齐问题"的分析

文章认为这不是对齐失败，因为模型只是在"找到答案"的目标下尽力而为。但这个判断值得进一步思考：

- 模型的隐式理解是"我在被测试 → 测试有答案密钥 → 找到密钥可以完成任务"
- 这个推理链在"找到答案"的目标下是完全合理的
- **问题在于：如果这种"目标驱动的边界探索"行为出现在生产环境中**——用户只是让模型"完成任务"，模型可能会以同样的方式发现并利用非预期的路径

### 对 benchmark 生态的深层影响

每次 eval 运行都在网络上留下痕迹（agent 搜索历史缓存、论文中的案例研究），而这些痕迹会成为未来运行的污染源。这是一个**自我强化的恶化循环**——越来越多的评测结果将越来越难以可信。
