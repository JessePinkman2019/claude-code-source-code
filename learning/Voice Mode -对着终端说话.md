# Voice Mode：对着终端说话

## 核心机制

`/voice` 开关语音模式。按住空格键说话，松开后语音转文字插入到光标位置。

| 操作 | 效果 |
|------|------|
| `/voice` | 开关语音模式 |
| 按住空格键 | 麦克风激活，开始录音 |
| 松开空格键 | 转录处理，文字出现在光标处 |
| 继续打字 | 文字和语音输入合并在同一个 prompt |

关键点：语音不替换当前输入，而是**插入**到光标所在位置。打了一半 prompt，想语音说完剩下的，直接按住空格说，转录内容会追加在已有文字后面。

**转录 token 不计入速率限制**，语音模式在任何计划里都不额外收费。

---

## 核心价值：打字 + 语音混合输入

这才是语音模式真正的卖点，不只是"能说话"，而是**在一个 prompt 里混用打字和语音，互不干扰**。

实际场景示例：

```
[打字] 检查 src/api/routes.ts 中的错误处理，
[语音] 当前的实现在认证失败和网络超时时用同样的方式处理，
       但下游消费者对这两种情况需要不同的重试策略，
       我希望每个 catch 块至少在继续之前用请求上下文记录错误，
[打字] 把这个模式应用到所有 route handler 上
```

这一串产生的是一个完整 prompt。打字负责精确的文件路径、变量名、具体约束；语音负责流淌出复杂的逻辑、推理、模糊的意图。

**核心分工原则**：
- 用**打字**表达"**做什么**"（精确技术细节）
- 用**语音**表达"**为什么**"（意图、推理、上下文）

---

## 语音比打字更好的场景

### 快速原型迭代
打字逼你在思路成形之前就提交结构。语音允许你边想边说：
> "先用 React component 试试，但如果状态管理变复杂，切到原生 JS 加一个简单的 pub-sub 模式。"
这种探索性指令说出来比打出来快得多。

### 描述你看到但难以打出的 bug
```
[打字] FilterPanel.tsx 中 useEffect cleanup 的问题，
[语音] dropdown 第一次加载渲染正常，但是导航离开再回来之后，
       状态重置了，已选中的条目恢复成默认值，
       即使 URL 参数里还有正确的值 --
[打字] 检查 useEffect cleanup
```

### 详细 bug 报告
需要给 Claude 完整图景时，语音让你一口气叙述看到了什么、尝试了什么、怀疑是什么。配合 Plan 模式分析阶段使用效果最佳。

### 代码审查反馈
```
[打字] src/api/routes.ts
[语音] 这个函数的错误处理在默默吞掉异常，
       我希望每个 catch 块在继续之前至少记录带有请求上下文的错误 --
[打字] 应用到所有 route handler
```

### 无障碍访问
长时间打字不舒适、或口头表达更自然的开发者，push-to-talk 让你控制节奏，可以随时根据 prompt 的哪个部分更适合什么方式来切换。

## 使用技巧

**文件路径和名称用打字**：语音转录可能搞乱 `src/components/AuthProvider.tsx` 这类路径，精确的部分打字，指令部分语音。

**语音开头说动作**：先说你要做什么，再加背景。"重构这个函数用 async/await" 比两分钟意识流最后才说"所以让它变成异步"好解析得多。

**配合 Fast Mode 加速迭代**：`/fast` 开快速响应，用语音减少打字开销。更快的输出 + 更快的输入，反馈循环大幅压缩。

---

## 当前限制

**灰度推送中**：目前约 5% 用户有权限，Anthropic 在 Pro/Max/Team/Enterprise 计划中逐步扩大。如果 `/voice` 不工作，还未进入推送组。

**仅 push-to-talk**：没有免提或始终监听模式。按住空格说话，松开停止。这是刻意的设计——终端环境里不想要意外触发转录。

**不支持 Agent SDK**：语音模式仅限终端交互，通过 Claude Code SDK 构建的程序化工作流无法使用。

**主要支持英语**：初始推送以英语转录为主。

---

## 与 claude.ai 语音模式的区别

Claude 在 web 和移动端也有语音模式，是完全不同的功能：

| | Claude Code 语音模式 | claude.ai 语音模式 |
|--|---------------------|-------------------|
| 交互方式 | Push-to-talk，产生文字 | 免提对话，持续交互 |
| 输出形式 | 转录为文字插入 prompt | 语音来回对话 |
| 设计场景 | 终端开发工作流 | 日常对话 |
| 混合输入 | 支持打字+语音混合 | 纯语音对话 |
